🖥 Can You Run Kimi K2 Locally?

Kimi K2 is a 1 Trillion parameter MoE model. The smallest usable quantized version (1.8-bit UD-TQ1_0) is 245 GB on disk and needs ~256 GB combined RAM + VRAM for decent speed. Here's what that means for your hardware:

Your Machine Kimi K2 (1T) Best Local Alternative
MacBook M1/M2 — 8GB ✕ Impossible Qwen2.5:7b, Llama3.2:3b (snappy)
MacBook M2/M3 — 16GB ✕ Impossible Qwen2.5:14b, Mistral:7b
MacBook M3 Pro — 36GB ✕ Impossible Qwen2.5:32b ← great choice
Mac Studio M2/M3 Max — 64GB ✕ Impossible Qwen2.5:72b, Llama3.3:70b ← excellent
Mac Studio M2 Ultra — 192GB ~ Disk-offload only (<1 tok/s) Llama3.3:70b, Mistral-Large (fast)
Server — 256GB RAM + A100 80GB ~ ~1–2 tok/s (usable) Kimi K2 1.8-bit quant via llama.cpp
Server — 4× H200 (320GB VRAM) ✔ 40+ tok/s — full experience Kimi K2.5 Q4/Q5 quant

💡 Recommendation for most users: Use Ollama + Qwen2.5:72b (64GB Mac) or Qwen2.5:32b (36GB Mac). They are fast, excellent personal assistants, and beat many cloud models. Connect this bot to Ollama's OpenAI-compatible API at localhost:11434/v1.

🦙 Option A — Ollama (Easiest, Recommended)

Works on Mac, Linux, Windows. Manages models automatically. OpenAI-compatible API built-in.

  1. Download & install Ollama from ollama.com/download
  2. Start Ollama with CORS enabled (required for browser access):
    OLLAMA_ORIGINS=* ollama serve
  3. Pull your chosen model (run this once, it downloads the weights):
    # Best for 64GB Mac (M2/M3 Max/Ultra): ollama pull qwen2.5:72b # Best for 32-36GB Mac: ollama pull qwen2.5:32b # Kimi K2 via Ollama cloud routing (no local weight download): ollama pull kimi-k2:1t-cloud # Kimi K2.5 multimodal via cloud routing: ollama pull kimi-k2.5:cloud
  4. In Bol-ZombAI accordion → switch to Local AI mode
  5. Set endpoint: http://localhost:11434/v1
  6. Select your model from the dropdown and speak!
ℹ Note on kimi-k2:1t-cloud — this routes through Ollama's cloud servers, not your machine. Your prompt leaves your device. For truly 100% local, use qwen2.5 or llama3.3.

⚙ Option B — llama.cpp Server (For Power Users)

For actual Kimi K2 GGUF weights running fully locally on high-end hardware.

  1. Clone and build Unsloth's llama.cpp fork (required for Kimi K2):
    git clone https://github.com/unslothai/llama.cpp cmake llama.cpp -B llama.cpp/build \ -DBUILD_SHARED_LIBS=OFF -DGGML_CUDA=ON -DLLAMA_CURL=ON cmake --build llama.cpp/build --config Release -j \ --target llama-server llama-cli
  2. Download quantized Kimi K2 from Hugging Face (245 GB minimum):
    huggingface-cli download \ unsloth/Kimi-K2-Instruct-GGUF \ "UD-TQ1_0/Kimi-K2-Instruct-UD-TQ1_0-00001-of-00005.gguf" \ --local-dir ./kimi-k2
  3. Start the OpenAI-compatible server with CORS:
    LLAMA_SET_ROWS=1 ./llama.cpp/build/bin/llama-server \ --model ./kimi-k2/UD-TQ1_0/Kimi-K2-Instruct-UD-TQ1_0-00001-of-00005.gguf \ --alias "kimi-k2-local" \ -ot ".ffn_.*_exps.=CPU" \ --ctx-size 16384 \ --port 8001 \ --cors
  4. In Bol-ZombAI accordion → Local AI mode
  5. Set endpoint: http://localhost:8001/v1, model: kimi-k2-local

🎙 Voice Output — 100% Local (No ElevenLabs)

  • Leave ElevenLabs key empty — the bot will fall back to the browser's built-in Web Speech Synthesis API automatically (no internet needed for TTS either)
  • For higher quality local TTS: run Kokoro-TTS or Coqui TTS locally and point the bot at it
  • Browser Speech Synthesis works on all platforms, is instant, and completely offline

🍎 Deploy on MacBook

  1. Save this file anywhere — e.g. ~/Desktop/bol-zombai.html
  2. Open Terminal (Cmd+Space → "Terminal")
  3. Navigate to the folder: cd ~/Desktop
  4. Start a local server: python3 -m http.server 8080
  5. Open browser → http://localhost:8080/bol-zombai.html
  6. Allow microphone permissions when prompted 🎤
  7. Paste your API keys in the config accordion — they stay in memory only

🔑 Getting Your API Keys

  • Anthropic Key: console.anthropic.com → API Keys → Create Key. Starts with sk-ant-
  • ElevenLabs Key: elevenlabs.io → Profile → API Key section
  • Voice ID: ElevenLabs → Voices → click voice → copy ID from URL. Default is "Rachel"
  • Keys are entered in the UI and only used in-browser — never stored on disk

⚡ Quick Notes

  • Zero dependencies — single HTML file, works anywhere
  • Best on Chrome or Edge (strongest mic / Speech API support)
  • Firefox works for chat; mic features need a local server due to browser security

📜 How To Run Bol-ZombAI

  1. Click the accordion header to expand API Keys & Persona config
  2. Enter your Anthropic API Key
  3. Enter your ElevenLabs API Key
  4. Optionally change the Voice ID to another ElevenLabs voice
  5. Choose a Persona — Bol-ZombAI speaks as an undead oracle by default
  6. Click the red 🎤 SPEAK button — allow mic permission when asked
  7. Speak your question clearly into the mic
  8. Click 🛑 STOP when done speaking
  9. Bol-ZombAI transcribes → sends to Claude → speaks back via ElevenLabs 🔊
  10. The audio player appears and auto-plays the undead response!
