6 models compared

            


            

              

                Budget testing
                MiniMax Hailuo Fast and lower-resolution Runway-style tests are better when you need fast learning loops without burning credits.
              

              

                Premium motion
                Runway Gen-4 and Veo-class models are stronger when you care about polished motion, composition, and presentation quality.
              

              

                Storyboard continuity
                Seedance and Kling-style models are useful when the script depends on multi-shot flow, stylized control, or stronger cinematic continuity.

Key pipeline layers in code

                
The file is structured like an orchestrator, not a single generator call. It separates planning, media generation, and execution controls into reusable blocks.

                
1. Environment + config
   - load_default_env()
   - getenv_int()
   - configure_logging()

2. Brief normalization
   - get_character_names()
   - get_main_character_names()
   - get_provided_master_character_prompts()
   - get_character_reference_images()

3. Narrative generation
   - OpenAIResponsesClient.generate_series_bible()
   - OpenAIResponsesClient.generate_episode_package()

4. Media generation
   - OpenAIImageClient
   - CommandTemplateRunner
   - provider routing for Runway / Pika / Veo

5. Execution modes
   - package
   - package-image
   - full
   - image-only
   - video-only

6. Delivery
   - episode JSON
   - scene JSON
   - prompts
   - rendered image/video outputs
   - logs and reports

OpenAI image generation with GPT Image

                
Endpoint: POST /v1/images/generations
Use case: Generate an image from a prompt using gpt-image-1.

                
Request
POST https://api.openai.com/v1/images/generations
Authorization: Bearer $OPENAI_API_KEY
Content-Type: application/json

{
  "model": "gpt-image-1",
  "prompt": "A cinematic product shot of a futuristic electric bicycle in a clean studio, soft blue accent lighting, premium commercial style",
  "size": "1536x1024",
  "quality": "medium"
}

Sample response
{
  "created": 1740000000,
  "data": [
    {
      "b64_json": "iVBORw0KGgoAAAANSUhEUgAA..."
    }
  ]
}

              


              

                
Runway image-to-video generation

                
Endpoint: POST /v1/image_to_video
Use case: Animate a still image into a short video clip with prompt-guided motion.

                
Request
POST https://api.dev.runwayml.com/v1/image_to_video
Authorization: Bearer $RUNWAY_API_KEY
X-Runway-Version: 2024-11-06
Content-Type: application/json

{
  "model": "gen4.5",
  "promptImage": "https://example.com/hero-frame.png",
  "promptText": "A gentle cinematic push-in, subtle subject motion, soft atmospheric background movement",
  "ratio": "1280:720",
  "duration": 5
}

Sample create-task response
{
  "id": "17f20503-6c24-4c16-946b-35dbbce2af2f",
  "status": "PENDING"
}

Sample completed task response
GET /v1/tasks/17f20503-6c24-4c16-946b-35dbbce2af2f

{
  "id": "17f20503-6c24-4c16-946b-35dbbce2af2f",
  "status": "SUCCEEDED",
  "output": [
    "https://dnznrvs05pmza.cloudfront.net/....mp4"
  ]
}

              

            

            

              OpenAI image generation commonly returns base64 image payloads, while Runway video generation returns a task id first and the final video URL after task completion.
